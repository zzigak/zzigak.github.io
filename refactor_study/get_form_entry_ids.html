How to Get Entry IDs from Your Google Form

Method 1: Using Browser Developer Tools (Recommended)

  1. Open your Google Form in edit mode
  2. Click the "Preview" button (eye icon) to open the form
  3. Right-click on the form and select "Inspect" or press F12
  4. Go to the "Network" tab in Developer Tools
  5. Fill out the form with test data and submit it
  6. Look for a request to "formResponse" in the Network tab
  7. Click on it and go to the "Payload" or "Form Data" tab
  8. You'll see all the entry IDs like: entry.1958938177

Method 2: View Page Source

  1. Open your Google Form (the public view, not edit mode)
  2. Right-click and select "View Page Source" or press Ctrl+U (Cmd+U on Mac)
  3. Search for "entry." in the source code
  4. You'll find patterns like:
    <input type="hidden" name="entry.1958938177" value="">
  5. Match each entry ID with its corresponding field by looking at the labels nearby

⚠️ Important Notes:

  • Entry IDs are unique to each form and cannot be guessed
  • The order of fields in the form matches the order of entry IDs
  • Radio buttons (like V1/V2 choice) will have the same entry ID with different values
  • For radio buttons, the value you send should match exactly what's shown in the form (e.g., "V1" or "V2")

Expected Entry IDs for Your Form Structure:

Based on your form structure, you should find these entry IDs:

// User Information entry.XXXXXXXXX - UserID entry.XXXXXXXXX - Email entry.XXXXXXXXX - Name // Trial 1 entry.XXXXXXXXX - TupleID (Trial 1) entry.XXXXXXXXX - PairID (Trial 1) - Radio: 1, 2, or 3 entry.XXXXXXXXX - Choice (Trial 1) - Radio: V1 or V2 // Trial 2 entry.XXXXXXXXX - TupleID (Trial 2) entry.XXXXXXXXX - PairID (Trial 2) - Radio: 1, 2, or 3 entry.XXXXXXXXX - Choice (Trial 2) - Radio: V1 or V2 // ... continue for Trials 3-10 ... // Additional fields entry.XXXXXXXXX - Trial Number entry.XXXXXXXXX - Timestamp

Quick Test Script

Once you have the entry IDs, you can test with this script in the browser console:

// Run this in the console while on your Google Form page const inputs = document.querySelectorAll('[name^="entry."]'); const entryIds = {}; inputs.forEach(input => { const label = input.closest('.freebirdFormviewerComponentsQuestionBaseRoot') ?.querySelector('[role="heading"]')?.textContent || 'Unknown'; entryIds[label] = input.name; }); console.table(entryIds);

After Getting the Entry IDs

  1. Replace all entry.XXXXXXXXX placeholders in study_single_form.js with the actual entry IDs
  2. Test with one complete submission to ensure all fields are mapped correctly
  3. The pre-filled URL will then work properly when users complete the study
